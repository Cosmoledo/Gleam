# audit-src

One unified audit of `src/`. Reports only deviations — never "compliant" entries, never a resolved-issues log.

## Arguments

- (none) — print findings to chat
- `file` — write findings to root as `AUDIT.md` (gitignored) instead of chat; overwrites prior contents

## Files to skip

- `src/loader/TiledMap.ts`

All other files under `src/**/*.ts` are in scope.

## Rules to enforce

`AGENTS.md` is the binding rules document. Read it first. Audit against these sections of `AGENTS.md`:

- **A.** § _Coding conventions_ — pay special attention to the JSDoc rule: flag any public export (class, free function, public method/accessor, exported interface/type, or its fields) that has no JSDoc, JSDoc that omits info the type can't carry (defaults, units, ranges, semantic constraints), or JSDoc that contradicts the implementation (wrong param name, stale return shape, wrong default, behavior the code no longer does).
- **B.** § _Class-member order_ (and its exceptions: `draw` first, `update` second; RGBA order on color channels; `Rect` xywh+sides grouping)
- **C.** § _Prototype augmentation_ (including its _For CanvasRenderingContext2D specifically_ subsection)

Plus one catch-all category not in AGENTS.md:

- **D. General correctness** — obvious bugs, dead branches, hot-path allocations in documented/possible hot loops, broken public API contracts, type-safety holes (`as any`, unchecked non-null, etc.). Only list concrete, verifiable issues — no speculation.

## Process

1. **Read `AGENTS.md`** to confirm the binding rules.
2. **Enumerate eligible files** — `src/**/*.ts` minus the skip list.
3. **Scan in parallel** — when more than ~10 files remain, spawn subagents (one per directory group, or batched by ~8 files each) with a focused brief: the file list it owns, the rule sections it should check, and the requirement to return findings as `filename:line — issue — Severity: Low|Medium|High`.
4. **Merge findings** from all scans, dedupe, sort by file path then line.
5. **Output** per the argument:
    - default → print findings to chat (see format below).
    - `file` → write to `AUDIT.md` and print only the summary line to chat.

## Output format

Group by rule section (A / B / C / D). Within each section, one line per finding. Each finding is prefixed with a 1-based index that **keeps incrementing across sections** — section B does not restart at 1:

```
1. src/foo/Bar.ts:42 — uses for-of where forEach would do — Severity: Low
2. src/foo/Bar.ts:88 — JSDoc omits default `count = 0` — Severity: Low
3. src/baz/Qux.ts:17 — instance accessor below constructor — Severity: Medium
```

(Numbers above are illustrative — `1`/`2` would sit in section A, `3` in section B, and so on.)

Sections with no findings are omitted entirely. If nothing deviates anywhere, output exactly `No deviations found.` and stop.

When writing to `AUDIT.md`, use this top matter:

```
# src/ audit

Generated by /audit. Lists only deviations from rules in AGENTS.md.

## Findings
```

…followed by the same grouped listing.

## Constraints

- **Don't auto-fix.** This skill audits only. Wait for explicit `go ahead` / `proceed` / `apply` before editing source (per `AGENTS.md` § _Analysis workflow_).
- **Don't list compliant findings or a fixed-issues log** — only current deviations.
- **Don't commit `AUDIT.md`**
